> 점검 관점: QA 엔지니어 기능 시나리오 테스트  
> 대상 시점: 프롬프트 3 완료 직후  
> 점검 방법: `script.js` 정적 코드 분석 + 예상 동작 도출 (브라우저 수동 테스트 권장)

| QA-10 | 게임 오버 | 1. 블록을 맨 위까지 쌓아 스폰 위치가 막힐 때까지 플레이 2. 낙하·조작 반응 확인 | 스폰 불가 시 게임 정지, 낙하·키 입력 무시 | ⚠️ Partial — `stopGame()` 호출되나 **게임 오버 UI·메시지 없음**, 겹친 블록이 보일 수 있음 |
| QA-11 | 다시 시작 | 1. 게임 진행 또는 게임 오버 후 **재시작** 클릭 2. 보드·점수·낙하·조작 확인 | 보드 초기화, 점수 0, 새 블록 스폰, 낙하·키보드 재동작 | ✅ Pass — `startGame()` → `resetGame()` + 타이머 재시작 |

## 실패·미구현 항목 상세

### QA-08 라인 클리어 (미구현)

- 완성 행 검사·제거·보드 `unshift` 로직 없음
- 블록이 계속 누적되어 QA-10 게임 오버로 빠르게 도달

### QA-09 점수 증가 (미구현)

- `#score` 요소는 존재하나 `resetGame()` 시 `0`으로만 설정
- 라인 클리어·hard drop 보너스 등 점수 규칙 없음

### QA-10 게임 오버 (Partial)

**구현된 부분:**
```javascript
if (!canMove(currentPiece, 0, 0, board)) {
  stopGame();
}
```

**미구현 부분:**
- "Game Over" 메시지·오버레이 없음
- 플레이어가 종료 인지 어려움
- 겹친 스폰 블록이 화면에 남을 수 있음
